**Status:** TENDING
**Date:** 2026-04-02
**Source:** Unresolved prompts (March 29-31 Reconciliation)

## The "Endless Box" Model

The IRF instrument is suffering from ID assignment collisions and is becoming too rigid. v3 should be a more fluid, high-velocity intake.

### Key Concepts
- **Endless Box:** A receiving mechanism for "slips of paper" (status updates, new tasks, context fragments).
- **Pencil Pushers:** Worker agents that take raw intake and organize it into the canonical index.
- **Unique IDs:** A standardized ID system across all 8 organs (no more collisions).

### Governance Rules
- **No Reduction of GitHub Issue Fields:** IRF is the staging ground, but GitHub remains the record of truth for work in progress.
- **Trust Record Keeper:** A role for an agent (or human) to verify the integrity of the collected box.
